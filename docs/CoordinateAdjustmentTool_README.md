# 座標調整工具使用說明

## 📋 工具概述

`CoordinateAdjustmentTool.ahk` 是一個專為《深空之眼》遊戲設計的視覺化座標調整工具，用於快速校正圖片辨識範圍，避免因遊戲畫面更新導致的座標偏移問題。

## 🎯 主要功能

- **視覺化調整**：在遊戲視窗上直接顯示辨識區域的紅色覆蓋框
- **拖拽操作**：支援滑鼠拖拽調整辨識區域位置
- **即時預覽**：選擇項目時顯示對應的辨識圖片
- **配置保存**：將調整後的座標保存為JSON格式
- **批量管理**：支援同時調整所有17個辨識項目

## 📁 支援的辨識項目

### 通用項目

- **戰鬥判定** - 判斷當前是否處於戰鬥狀態
- **烤肉紅判定** - 烤肉模式紅色能量判定
- **烤肉藍判定** - 烤肉模式藍色能量判定

### 魂羽角色

- **魂羽F1** - F技能判定區域1

## ⌨️ 操作說明

### 基本操作

1. **啟動工具**：雙擊執行 `CoordinateAdjustmentTool.ahk`
2. **進入調整模式**：按 **F8** 鍵進入調整模式
3. **調整座標**：在遊戲視窗中拖拽紅色框調整位置
4. **保存配置**：按 **F9** 保存調整後的座標
5. **退出工具**：按 **Esc** 退出

### 詳細操作步驟

#### 步驟1：準備工作

- 確保遊戲《深空之眼》已啟動並處於視窗模式
- 遊戲視窗大小應為 1600×900（可使用主腳本的F2鍵調整）

#### 步驟2：啟動調整工具

- 雙擊執行 `CoordinateAdjustmentTool.ahk`
- 工具會顯示啟動提示："座標調整工具已啟動，按 F8 進入調整模式"

#### 步驟3：進入調整模式

- 按 **F8** 進入調整模式
- 工具會檢查遊戲視窗是否存在
- 如果找到視窗，會顯示調整介面並在遊戲視窗上疊加紅色辨識區域框

#### 步驟4：調整辨識區域

- 在調整工具介面中選擇要調整的項目
- 觀察遊戲視窗中的紅色框位置
- 使用滑鼠在遊戲視窗中拖拽紅色框到正確位置
- 調整工具介面會即時顯示當前座標和對應圖片

#### 步驟5：保存配置

- 按 **F9** 保存調整後的座標
- 配置會保存為 `config/coordinates_config.json` 文件
- 工具會顯示保存成功的提示訊息

#### 步驟5：保存配置
- 按 **F9** 保存調整後的座標
- 配置會保存為 `config/coordinates_config.json` 文件
- 工具會顯示保存成功的提示訊息

## 🖥️ 介面說明

### 調整工具介面
- **辨識項目列表**：顯示所有可調整的辨識項目
- **當前座標顯示**：顯示選中項目的X1,Y1,X2,Y2座標
- **圖片預覽區域**：顯示對應的辨識圖片
- **操作按鈕**：
  - 重新載入：重新載入座標配置
  - 保存配置：保存當前調整的座標
  - 刷新覆蓋：重新整理遊戲視窗上的紅色框
  - 退出：退出調整模式

### 遊戲視窗覆蓋
- **紅色半透明框**：表示當前辨識區域
- **即時更新**：拖拽時框會跟隨滑鼠移動
- **多區域同時顯示**：所有項目都會顯示對應的框

## 📄 配置文件

### config/coordinates_config.json 格式
```json
[
  {
    "name": "戰鬥判定",
    "x1": 88,
    "y1": 853,
    "x2": 150,
    "y2": 888
  },
  {
    "name": "魂羽F1",
    "x1": 1045,
    "y1": 684,
    "x2": 1565,
    "y2": 880
  }
]
```

### 配置文件位置
- 預設保存位置：`config\coordinates_config.json`
- 如果文件不存在，工具會使用內建的預設座標

## ⚠️ 注意事項

### 使用前準備
- 確保遊戲處於正確的視窗模式（1600×900）
- 關閉主腳本的自動功能，避免干擾調整
- 調整時應處於遊戲的技能可見狀態

### 調整技巧
- 優先調整戰鬥判定和常用技能
- 每次調整一個項目，確認位置正確後再調整下一個
- 可以使用遊戲的暫停功能（F4）來獲得更好的調整環境

### 常見問題
- **看不到紅色框**：檢查遊戲視窗是否被其他視窗遮擋
- **拖拽無效**：確保點擊在紅色框內部
- **座標不正確**：檢查遊戲解析度和視窗大小是否正確

## 🔄 更新流程

當遊戲畫面更新時的完整調整流程：

1. 啟動遊戲並調整到正確視窗大小
2. 執行座標調整工具
3. 進入調整模式，觀察哪些區域位置不正確
4. 逐一調整每個偏移的辨識區域
5. 保存配置並測試主腳本功能
6. 如有需要，重複調整直到所有功能正常

1. 檢查 `config/coordinates_config.json` 文件格式是否正確
